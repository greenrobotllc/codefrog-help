# Keyboard Shortcuts

## Global Shortcuts

| Shortcut | Action |
|----------|--------|
| `Ctrl+N` | New File… |
| `Ctrl+Shift+N` | New Project… |
| `Ctrl+Alt+N` | New Folder… |
| `Ctrl+W` | Close Window |
| `Ctrl+Shift+P` | Quick Project Switcher |
| `Ctrl+,` | App Settings |
| `Ctrl+R` | Refresh Current View |
| `Ctrl+F` | Find in Text Fields |
| `Ctrl+?` | Show Keyboard Shortcuts |
| `Alt+F4` | Quit CodeFrog |

| Shortcut | Action |

| `Alt+Right` | Go Forward |

## Terminal Shortcuts

| Shortcut | Action |
|----------|--------|
| `Esc` | Send Escape Key |
| `Ctrl+C` | Send Ctrl+C (Interrupt) |
| `Ctrl+D` | Send Ctrl+D (EOF) |
| `Tab` | Send Tab Key (Autocomplete) |
| `Ctrl+K` | Clear Terminal |
| `Ctrl+C` | Copy Selected Text (when text is selected) |
| `Ctrl+V` | Paste Text |

*Note: Terminal keyboard controls (Esc, Ctrl, Tab) work as expected for terminal interaction. When text is selected in the terminal, `Ctrl+C` copies to clipboard. When no text is selected, `Ctrl+C` sends the interrupt signal to the terminal.*

| Shortcut | Action |

## Keyboard Shortcut Tips

- **Discover Shortcuts:** Press `Ctrl+?` anytime to see available shortcuts
- **Context-Aware:** Some shortcuts change based on which tab or view is active
- **Terminal Focus:** When the terminal is focused, most shortcuts are passed through to the terminal
- **Windows Terminal:** If using Windows Terminal, it has its own shortcuts (see Windows Terminal settings)
- **Customization:** Keyboard shortcuts cannot be customized at this time

## Differences from macOS

On Windows, keyboard shortcuts use:
- **Ctrl** instead of **⌘ (Command)**
- **Alt** instead of **⌥ (Option)**
- **Shift** remains the same
- **F2** for rename (instead of Enter on macOS)
- **Alt+F4** to quit (instead of ⌘Q)
